Image segmentation is the task of partitioning a digital image into meaningful regions at pixel level, so that each pixel is assigned to a class or to a specific object instance.\cite{ibm_what_nodate} \\
Depending on the task formulation, segmentation can be divided into:
\begin{itemize}
    \item \textbf{Semantic segmentation}: assigns each pixel in the image to a predefined category, without distinguishing between different object instances belonging to the same class.

    \item \textbf{Instance segmentation}: detects and segments each individual object instance in the image, producing a separate mask for each countable object (typically referred to as \emph{things}), while usually not distinguishing amorphous background regions such as sky or ground (\emph{stuff}).

    \item \textbf{Panoptic segmentation}: unifies semantic and instance segmentation by providing a complete scene understanding, where both individual object instances (\emph{things}) and uncountable background regions (\emph{stuff}) are segmented in a consistent and non-overlapping manner.\cite{ibm_what_2023}
\end{itemize}

This fine-grained representation allows for a more accurate description of object shapes and boundaries than object detection, but it comes at the cost of higher computational complexity and increased sensitivity to noise, texture variability, and illumination changes.

Image segmentation can be addressed through both learning-based and classical image-processing approaches. Among the latter, clustering methods and background-subtraction techniques are often used to isolate regions on the basis of visual or geometric differences with respect to the surrounding scene.\cite{mittal_comprehensive_2022,opencv_background_subtraction_2026}

From an evaluation perspective, segmentation is typically assessed through pixel-level overlap metrics. The most common ones include \textbf{Intersection over Union (IoU)}, the \textbf{Dice coefficient}, and task-specific extensions such as \textbf{mean IoU (mIoU)} for semantic segmentation, mask-based \textbf{Average Precision (AP)} for instance segmentation, and \textbf{Panoptic Quality (PQ)} for panoptic segmentation.

These metrics are discussed in more detail in the evaluation metrics section, where the criteria adopted for object detection and segmentation are presented in a unified way.
