Site 1 represents the first deployment environment of the proposed vision-based monitoring system. This site corresponds to the production stage where the dough exits the industrial forming machine in the shape of individual cylindrical dough balls. The objective at this stage is twofold: (i) to acquire a representative RGB-D dataset for training the object detection network in an offline setting, and (ii) to deploy online the trained DEIMv2 to detect the bounding boxes containing the dough for volumetric estimation of the produced dough balls.

\subsubsection{Industrial Context and Process Description}
At Site 1, the dough mixture is processed by an automated forming machine that portions and shapes the material into approximately cylindrical dough balls. These dough balls exit the machine sequentially and move along a conveyor belt, until they reach the output table (Figure \ref{fig:Output}). Variability in size and shape may occur due to fluctuations in the upstream mixing and forming processes. For quality control purposes, it is essential to monitor geometric properties such as volume in a non-contact manner.

To address this requirement, a vision-based acquisition setup has been installed above the machine output (see Figure ~\ref{fig:site1}).
The hardware configuration consists of an \emph{NVIDIA Jetson Nano 2GB Developer Kit} \cite{nvidia_corporation_jetson_nodate} for edge computation and an \emph{Azure Microsoft Kinect} RGB-D sensor \cite{microsoft_azure_nodate} for synchronized color and depth acquisition. The Kinect is based on Time-of-Flight (ToF) technology, which enables dense depth estimation by measuring the phase shift of modulated infrared light \cite{microsoft_azure_nodate}.

\subsubsection{Offline Data Acquisition and Network Training}
During the offline phase, the system operates in data acquisition mode. In this configuration, the Azure Kinect continuously captures synchronized RGB and depth frames. A motion detection algorithm is executed on the Jetson Nano to identify temporal variations in the scene corresponding to the passage of dough balls under the camera.

The motion detection module serves exclusively as a trigger mechanism for dataset collection. When motion is detected within a predefined and configurable region of interest (ROI), the corresponding RGB-D frame is stored. In this case, the chosen ROI completely coincided with the machine output. \\
This approach ensures that:
\begin{itemize}
    \item Only relevant frames containing dough balls are acquired,
    \item Redundant background movement is discarded,
    \item The dataset remains balanced and computationally manageable.
\end{itemize}

The collected dataset is subsequently annotated using the Language Segment-Anything framework \cite{medeiros_luca-medeiroslang-segment-anything_2026}. This pretrained open-source model integrates instance segmentation capabilities with text-based prompts, enabling the generation of object-specific segmentation masks directly from natural language descriptions.

In the proposed pipeline, a textual prompt describing the dough balls is provided to automatically obtain pixel-level masks for each instance present in the image. The generated masks are then converted into corresponding bounding boxes, which are used as ground-truth annotations for training the DEIMv2\cite{huang_real-time_2026} object detection network in an offline environment.

This semi-automatic annotation strategy significantly reduces manual labeling effort while preserving annotation consistency across the dataset.\\
Lastly, the training phase is performed on a separate workstation equipped with higher computational resources.

\subsubsection{Online Inference and 3D Volume Estimation}

After training, the system is deployed online at Site 1 using the same hardware configuration (Jetson Nano + Azure Kinect). However, the software pipeline changes significantly.

In the online phase:
\begin{enumerate}
    \item The motion detection module is disabled.
    \item The trained DEIMv2\cite{huang_real-time_2026} object detection network runs directly on the Jetson Nano.
    \item For each detected dough ball, the network outputs bounding boxes in the RGB image.
    \item Each 2D bounding box is then projected into the corresponding depth map to extract the associated depth pixels. By combining RGB and depth information, the system reconstructs a local 3D point cloud of each dough ball. The RGB-D to point cloud conversion follows the standard pinhole camera model:
\end{enumerate}

\begin{equation}
X = \frac{(u - c_x) \cdot Z}{f_x}, \quad
Y = \frac{(v - c_y) \cdot Z}{f_y}, \quad
Z = D(u,v)
\end{equation}

where:
\begin{itemize}
    \item $(u,v)$ are pixel coordinates,
    \item $D(u,v)$ is the depth value,
    \item $(c_x, c_y)$ are the principal point coordinates,
    \item $f_x, f_y$ are the focal lengths in pixel units.
\end{itemize}

\subsubsection{System Objectives at Site 1}

The implementation at Site 1 serves as a proof-of-concept for an edge-based RGB-D inspection system capable of:

\begin{itemize}
    \item Automated dataset acquisition in industrial conditions,
    \item Real-time object detection on embedded hardware,
    \item 3D reconstruction from RGB-D data,
    \item Non-contact volumetric estimation of deformable food products.
\end{itemize}

The modular design adopted at Site 1 ensures that the same sensing and computational infrastructure can be reused across both offline training and online inference phases, thereby minimizing hardware complexity and deployment costs.
