Site 2 corresponds to the acquisition and inspection station dedicated to baked bread loaves. In contrast to Site 1, where RGB-D data are exploited to estimate volumetric properties of raw dough balls, Site 2 focuses on the visual analysis of the final baked product. The primary objective at this stage is the creation of a dedicated image dataset of baked loaves, followed by the training and deployment of the DEIMv2 object detection network for automated detection and color-based quality assessment.

\subsection{Industrial Context and Monitoring Objectives}

At Site 2, the product has completed the baking process and is transported along the production line as finished bread loaves. At this stage, geometric variability is less critical compared to visual characteristics such as crust color, browning uniformity, and surface appearance. These attributes are strongly correlated with baking quality and process parameters (e.g., temperature profile, baking time, and humidity).

The system deployed at Site 2 aims to:

\begin{itemize}
    \item Acquire a representative dataset of baked bread images,
    \item Train a dedicated DEIMv2 object detection model,
    \item Detect bread loaves in real time,
    \item Estimate crust color metrics for quality monitoring.
\end{itemize}

\subsection{Hardware Configuration}

Unlike Site 1, which relies on an RGB-D sensor and embedded GPU platform, Site 2 adopts a lightweight and cost-effective vision setup composed of:

\begin{itemize}
    \item A Raspberry Pi 5 for edge computation,
    \item A Raspberry Pi Camera Module 3 Wide for RGB image acquisition.
\end{itemize}


The Raspberry Pi Camera Module 3 Wide provides a large field of view, enabling coverage of the table where the loaves are baked (as shown in Figure \ref{fig:breadloaf}). Since volumetric estimation is not required at this stage, depth sensing is unnecessary, and a monocular RGB configuration is sufficient.
\subsection{Dataset Creation and Offline Training}

During the initial phase, the system operates in data acquisition mode. Images of baked bread loaves are captured under controlled lighting conditions to ensure consistency and minimize color distortions due to environmental variability.

The collected dataset is subsequently annotated and used to train the DEIMv2 \cite{huang_real-time_2026} object detection network in an offline environment with higher computational resources. The trained model learns to localize each bread loaf within the scene by predicting bounding boxes.

This dataset creation phase is fundamental to ensure robustness of the detection model against variations in loaf shape, crust texture, illumination changes, and partial occlusions.

\subsection{Online Detection and Color Estimation}

Once training is completed, the DEIMv2 model is deployed on the Raspberry Pi 5 for real-time inference. For each acquired RGB frame:

\begin{enumerate}
    \item The DEIMv2 network detects the bread loaves and outputs bounding boxes.
    \item The image regions corresponding to each bounding box are extracted.
    \item Color features are computed within each detected region.
\end{enumerate}

Instead of performing 3D reconstruction as in Site 1, the analysis at Site 2 focuses on colorimetric evaluation. The extracted regions are processed to compute statistical color descriptors in a selected color space (e.g., RGB, HSV, or CIE Lab). Metrics such as mean intensity, hue distribution, or lightness component can be used to quantify crust browning.

Formally, if $\mathcal{R}_i$ denotes the set of pixels within the $i$-th detected bounding box, the mean color value in the RGB space can be expressed as:

\begin{equation}
\bar{C}_i = \frac{1}{|\mathcal{R}_i|} \sum_{(u,v) \in \mathcal{R}_i} C(u,v)
\end{equation}

where $C(u,v)$ represents the color vector at pixel $(u,v)$ and $|\mathcal{R}_i|$ is the number of pixels in the region.

These color descriptors can be compared against predefined thresholds or statistical baselines to identify under-baked or over-baked products.

\subsection{Role of Site 2 in the Overall Architecture}

Site 2 complements the functionality of Site 1 by extending the inspection framework from geometric analysis of raw dough to visual quality assessment of the final baked product. While Site 1 leverages RGB-D sensing for volumetric estimation, Site 2 employs a monocular RGB setup for color-based evaluation.

Together, the two sites demonstrate the flexibility of the proposed DEIM-based architecture, which can be adapted to different sensing modalities and quality metrics while maintaining a consistent edge-computing paradigm.
